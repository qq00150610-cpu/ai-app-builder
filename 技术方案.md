# 云端AI应用开发打包平台 - 技术方案

## 一、产品定位
手机端一站式AI低代码应用开发平台，用户通过可视化操作+AI辅助，快速生成并打包APK应用。

## 三、AI接入方式

### 平台默认AI
- 平台提供基础AI能力（用户无需配置即可使用）
- 限额机制：免费用户每日10次AI生成，会员不限

### 用户自定义AI
支持两种模式：
1. **云端模型**：用户配置自己的API Key（OpenAI、Claude、通义千问、DeepSeek等）
2. **本地模型**：用户部署Ollama等本地模型，App通过局域网调用

配置项：
```
- API地址（endpoint）
- API Key
- 模型名称
- 温度/最大token等参数
```

## 五、技术选型

### 前端（手机App）
| 模块 | 技术方案 | 理由 |
|------|---------|------|
| 跨平台框架 | Flutter / React Native | 一套代码同时支持Android/iOS |
| 可视化编辑器 | 自研组件库 + 拖拽引擎 | 轻量、可控 |
| 代码编辑器 | Monaco Editor (WebView) | 成熟方案 |

### 后端
| 模块 | 技术方案 | 理由 |
|------|---------|------|
| 主语言 | Node.js / Python / Go | 按团队熟悉度选择 |
| API框架 | Express / FastAPI / Gin | 对应语言框架 |
| 数据库 | MySQL 8.0 | 事务性强、成熟稳定 |
| 缓存 | Redis | 会话管理、限流计数 |
| 对象存储 | 阿里云OSS / 腾讯云COS | APK分发、项目文件存储 |
| 消息队列 | RabbitMQ / Redis Queue | 编译任务异步处理 |

### 编译构建
| 方案 | 说明 | 成本 |
|------|------|------|
| 方案A：自建编译服务 | 搭建Android编译环境，接收项目配置动态生成代码并编译 | 服务器成本高，可控性强 |
| 方案B：调用第三方API | 集成AppGyver、Flutter Build等云编译服务 | 按次计费，成本可控 |
| 方案C：混合方案 | 简单应用用模板渲染，复杂应用调用第三方服务 | 平衡成本与能力 |

**推荐方案C**：初期用模板渲染快速验证，后期逐步自建编译能力。

### 支付
- 微信支付 / 支付宝
- 可考虑接入第三方聚合支付（如易支付）降低接入成本

## 七、数据库设计

### 用户表 (users)
```sql
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    phone VARCHAR(20) UNIQUE,
    wechat_openid VARCHAR(64) UNIQUE,
    nickname VARCHAR(50),
    avatar VARCHAR(255),
    member_type TINYINT DEFAULT 0, -- 0:免费 1:年费 2:永久
    member_expire_at DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 用户AI配置表 (user_ai_configs)
```sql
CREATE TABLE user_ai_configs (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    config_name VARCHAR(50),
    model_type VARCHAR(20), -- 'openai', 'claude', 'ollama', 'qwen'...
    api_endpoint VARCHAR(255),
    api_key VARCHAR(255),
    model_name VARCHAR(50),
    temperature DECIMAL(2,1) DEFAULT 0.7,
    max_tokens INT DEFAULT 2048,
    is_default TINYINT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 项目表 (projects)
```sql
CREATE TABLE projects (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    name VARCHAR(100),
    description TEXT,
    config JSON, -- 页面配置、组件配置
    status TINYINT DEFAULT 1, -- 1:编辑中 2:编译中 3:已完成
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 编译任务表 (build_tasks)
```sql
CREATE TABLE build_tasks (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    project_id BIGINT NOT NULL,
    user_id BIGINT NOT NULL,
    status TINYINT DEFAULT 0, -- 0:排队中 1:编译中 2:成功 3:失败
    apk_url VARCHAR(255),
    error_msg TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    completed_at DATETIME
);
```

### 订单表 (orders)
```sql
CREATE TABLE orders (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_no VARCHAR(32) UNIQUE,
    user_id BIGINT NOT NULL,
    product_type TINYINT, -- 1:年费 2:永久
    amount DECIMAL(10,2),
    pay_method VARCHAR(20),
    pay_status TINYINT DEFAULT 0, -- 0:待支付 1:已支付 2:已退款
    trade_no VARCHAR(64),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    paid_at DATETIME
);
```

## 十、风险与对策

| 风险 | 对策 |
|------|------|
| 编译服务不稳定 | 多节点部署，失败自动重试 |
| AI成本失控 | 用户自备API Key为主，平台限额兜底 |
| APK安全合规 | 应用签名隔离，内容审核机制 |
| 并发编译压力大 | 队列限流，会员优先 |
